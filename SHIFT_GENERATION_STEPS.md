# シフト作成ロジック - 実行ステップ一覧

現在 `ShiftEngine.js` で実行されているシフト作成プロセスの全ステップ（時系列順）です。
調整や修正を行う際は、このフローを参照してください。

| ステップID | 関数名 | 処理内容 | 備考 |
| :--- | :--- | :--- | :--- |
| **Step 0** | `step0_Initialize` | **初期化・データ正規化**<br>マスタデータを読み込み、キーの正規化やエイリアス統合（隠れデータ対策）を行いつつ、シフト表を初期化します。<br>※アプリ設定が最優先されるよう調整済み。 | 全ての日付を空文字 `""` で初期化。 |
| **Step 1** | `step1_ApplyCarryOver` | **繰越設定の適用**<br>前月から持ち越されたシフト（月初1～2日目など）を適用し、ロックします。 | 「繰越」シート使用。 |
| **Step 2** | `step2_ApplyPreferences` | **希望シフトの適用**<br>「希望休」「固定休」「出勤不可」「夜勤希望」などを反映し、ロックします。 | 「希望」シート使用。 |
| **Step 2.5** | `step2_5_ApplyFixedDays` | **曜日固定の適用**<br>「曜日固定」設定があるスタッフに対し、指定曜日以外を「休」にしてロックします。 | 専属かどうかにかかわらず適用。 |
| **Step 2.7** | `step2_7_ApplyRoleBasedFixedShifts` | **役職者シフトの固定**<br>施設長、サ責、事務員、管理者のシフトを決定します。<br>・公休数に合わせてランダムに「休」を配置。<br>・残りの空きマスを全て「予（予定）」で埋める。<br>・5連勤以上にならないよう調整。 | 対象：施設長, サ責, 事務員, 管理者 |
| **Step 2.8** | `step2_8_ForceAdjustRoleHolidays` | **役職者公休数の強制補正**<br>Step 2.7の後、公休数（設定値）に対して不足や過剰がある場合、強制的に「予」と「休」を入れ替えて数を合わせます。 | 「曜日固定」などで数が合わなかった場合の安全策。 |
| **Step 3** | `step3_ApplyWorkDaysPerWeek` | **週勤務日数の制限**<br>「勤務日数/週」の設定があるスタッフに対し、週の上限日数を超えないよう空き日を調整します。 | |
| **Step 4** | `step4_ApplyExclusive` | **専属スタッフの確定**<br>「専属」フラグがあるスタッフのシフトをここで完全に確定させ、ロックします。<br>※これ以降のステップでは変更されません。 | 対象：専属フラグがONのスタッフ |
| **Step 4.5** | `step4_5_ApplyTrainee` | **専属研修生の割り当て**<br>「専属」かつ「研修生」のシフトを決定します。<br>・研修（早/日/遅）をランダム配置。<br>・公休数調整と、連勤・シフト順序（遅→早禁止）の補正。 | 対象：専属 かつ 研修生 |
| **Step 5** | `step5_ApplyNightShift` | **夜勤の自動割り当て**<br>専属以外のスタッフ（研修生除く）に対し、必要人数を満たすように夜勤（夜→明→休）を割り当てます。 | 連続勤務制限や間隔を考慮。 |
| **Step 6** | `step6_ApplyEarlyLateShift` | **早番・遅番の割り当て**<br>各日の必要人数を満たすように、可能なスタッフ（研修生除く）に早番・遅番を割り当てます。 | |
| **Step 7** | `step7_ApplyDayLeader` | **日勤リーダーの割り当て**<br>「日L（日勤リーダー）」を割り当てます（研修生除く）。 | |
| **Step 8** | `step8_ApplyDayShift` | **日勤の割り当て**<br>残りの必要人数を正規スタッフの「日（日勤）」で埋めます。<br>※勤務回数の少ないスタッフを優先します。 | 研修生は対象外。 |
| **Step 8.2** | `step8_2_ForceFillDeficits` | **勤務日数不足の強制補充**<br>勤務日数が極端に少ないスタッフ（専属含む）に対し、空き日に強制的に「日」を割り当てます。 | **（新規追加）**<br>シフトが入らない特定のスタッフを救済。 |
| **Step 4.5** | `step4_5_ApplyTrainee` | **専属研修生の割り当て**<br>正規スタッフの割り当てが終わった後、「研修生」の空きシフトを決定します。 | Step 8.2の後に実行。 |
| **Step 8.5** | `step8_5_BalanceWorkDays` | **勤務日数の調整**<br>公休数に対して出勤日数が不足しているスタッフにシフトを追加し、多すぎる場合は減らします。<br>※専属スタッフも対象に含めるよう変更。 | 月間の公休数バランスを整える最終調整。 |
| **Step 8.8** | `step8_8_AdjustStaffCounts` | **要員数の強制調整**<br>各日付について、「早」「遅」「夜」「日」の人数が設定値を超えている場合、**勤務数が多いスタッフ**から優先的にシフトを削除します。 | 正規スタッフによる人員過剰を防止。 |
| **Step 9** | `step9_DistributeOffDays` | **休日の割り当て**<br>まだ埋まっていない全ての空きマス（`""`）を「休」で埋めます。 | これで全てのマスが埋まります。 |
| **Step 9.5** | `step9_5_ChiefAdjustment` | **主任による調整**<br>主任（Chief）の「予」を利用して、毎日の欠員補充とスタッフの休日バランス調整（交換）を強力に行います。 | **（強化）**<br>Step 9の後に実行することで、最終的なバランスを整える。 |
| **Step 10** | `step10_FinalizeValidation` | **最終確認**<br>作成されたシフト表の整合性をチェックし、ログを出力します。 | |

## 開発・調整時の注意点

*   **前のステップを壊さない**: 特定のステップを修正する際、その前後のステップ（特に既に確定したシフト）に影響を与えないようにしてください。
*   **「ロック」の尊重**: `isLocked: true` となっているセルは、いかなる場合も上書きしてはいけません。
*   **専属スタッフ**: Step 4 で確定した後も、Step 8.2以降の調整ロジック（不足補充）の対象となる場合があります（設定ミス対策のため）。
*   **研修生**: 正規スタッフの割り当て（Step 5～8）が完了してから（Step 4.5）シフトが割り当てられます。これにより、要員数カウントから完全に分離されています。
